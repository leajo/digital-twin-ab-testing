# TwinPilot — 디지털 트윈 기반 A/B 테스트 시뮬레이션 플랫폼

## 1. 프로젝트 소개

- **한 줄 설명:** 실제 고객 행동 데이터를 기반으로 디지털 트윈(가상 유저)을 생성하고, 실제 트래픽 없이 A/B 테스트 결과를 사전에 시뮬레이션하는 서비스

- **주요 기능:**
  - **3스텝 파이프라인:** 고객 데이터 셋팅 → 시나리오 입력 → 결과 리포트 (중간 단계 자동화)
  - **시나리오별 동적 세그먼트:** 시나리오 유형(프로모션, CTA 변경, 퍼널 변경 등)에 따라 주요 행동 변수를 자동 도출하고, 해당 변수 기반으로 유저 세그먼트를 동적으로 재구성
  - **동일 유저 대상 A/B 비교:** 동일한 디지털 트윈이 시나리오 A와 B를 모두 경험하여 유저 구성 차이 없이 순수한 시나리오 효과만 비교
  - **세그먼트 특성 기반 동적 반응 모델:** 세그먼트의 전환율·이탈률 등 행동 특성에 따라 variant별 반응 강도를 자동 차별화하여, 통계적으로 유의미한 세그먼트별 차이를 도출
  - **세그먼트 생성 과정 투명화:** 기본 세그먼트(전체 변수 기반) 상세 정보와 시나리오 맞춤 세그먼트(주요 변수 기반) 생성 과정을 단계별로 안내
  - **7개 섹션 결과 리포트:** 실험 요약, 핵심 지표 비교, 세그먼트별 전환율 히트맵, 태그별 분석, 퍼널 비교, 통계 검정(카이제곱, Cohen's h, 95% 신뢰구간), 세그먼트별 최적 시나리오
  - **자동 인사이트 생성:** 결과 데이터에서 핵심 인사이트를 규칙 기반으로 자동 도출 (세그먼트별 차별 적용 제안 포함)
  - **리포트 CSV 다운로드:** 시뮬레이션 결과를 CSV 파일로 내보내기
  - **업종별 샘플 데이터 + 맞춤 시나리오:** 패션 이커머스, 금융, OTT 콘텐츠 3개 업종의 샘플 데이터와 업종별 맞춤 시나리오 프리셋 제공

- **MVP 범위:**
  본 프로젝트는 **샘플 데이터 기반 데모 이용까지를 MVP로 구현**했습니다. 업종별 가상 고객 데이터를 자동 생성하고, 시나리오를 설정하여 시뮬레이션을 실행하고, 7개 섹션의 결과 리포트를 확인하는 전체 플로우를 체험할 수 있습니다. 실제 고객 데이터 연동 및 실시간 피드백 루프는 고도화 단계에서 구현 예정입니다.

## 2. 문제 정의

- **어떤 문제/불편을 해결하려 했는지:**
  - 기존 A/B 테스트는 통계적 유의성 확보까지 최소 2~4주가 소요되며, 실제 트래픽을 분할해야 하므로 매출 손실 리스크가 존재함
  - 충분한 샘플 확보를 위해 대규모 트래픽이 필요하여, 트래픽이 적은 서비스나 신규 기능에서는 A/B 테스트 자체가 어려움
  - 테스트 결과가 전체 전환율만 제공하여, "어떤 유저 그룹에서 효과적인지"에 대한 세그먼트별 인사이트가 부족함

- **기존에는 어떻게 처리했는지:**
  - 실제 트래픽을 분할하여 2~4주간 라이브 A/B 테스트 실행 (시간/비용 소모)
  - 트래픽 부족 시 감에 의존한 의사결정 또는 테스트 자체를 포기
  - 기존 시장의 대안(Synthetic Users, Mnemonic AI 등)은 LLM 기반 정성적 피드백/페르소나 인터뷰에 집중하여, 실제 행동 데이터 기반 정량적 시뮬레이션은 부재

## 3. AI(Kiro) 활용 방식

- **Kiro를 어떻게 활용했는지:**
  - 아이디어 구체화 단계부터 시장 조사, 기술 설계, 코드 구현, 테스트, 배포까지 전 과정을 Kiro와 협업하여 진행
  - 비개발자가 기획 의도를 자연어로 전달하면, Kiro가 기술 설계를 제안하고 코드를 구현하는 방식으로 개발 진행
  - 기획 방향에 대한 질문과 토론을 통해 MVP 플로우를 점진적으로 정제 (예: 세그먼트 태깅 기능 → 동적 재세그먼트 → 3스텝 파이프라인 → 하이브리드 세그먼트 전략)
  - 구현 후 한계점 분석 및 고도화 방향 논의까지 Kiro와 함께 수행

- **활용한 주요 기능:**
  - **Spec (스펙 기반 개발):** 요구사항 문서(15개 요구사항) → 기술 설계 문서(8개 컴포넌트, 21개 정확성 속성) → 구현 태스크(20개 태스크) 순서로 체계적 개발. 중간에 사용자 피드백을 반영하여 설계를 3차례 재구성
  - **대화형 기획:** 시장 조사(경쟁사 분석), MVP 범위 결정, 데이터 양식 설계, 테스트 시나리오 도출, 세그먼트 전략, LLM 활용 범위 논의, 서비스 한계 분석 및 고도화 로드맵 수립 등을 대화를 통해 진행
  - **코드 생성 및 디버깅:** Python 백엔드(FastAPI) 12개 서비스 모듈, React 프론트엔드, Streamlit 데모 앱을 Kiro가 생성. 배포 과정에서 발생한 모듈 경로 문제, CSS 적용 문제 등을 실시간으로 디버깅

## 4. 기대 효과

- **이 프로젝트가 가져올 변화:**
  - **A/B 테스트 사전 검증:** 실제 트래픽을 사용하기 전에 시뮬레이션으로 시나리오 효과를 예측하여, 실패 확률이 높은 시나리오를 사전에 걸러낼 수 있음
  - **의사결정 속도 향상:** 2~4주 걸리던 A/B 테스트 결과를 수 분 내에 시뮬레이션으로 확인 가능
  - **세그먼트별 차별 전략:** "전체적으로 B가 낫다"가 아니라 "가격 민감형에게는 A, 브랜드 충성형에게는 B"와 같은 세그먼트별 최적 전략 도출 가능
  - **트래픽 부족 서비스 지원:** 대규모 트래픽 없이도 디지털 트윈 기반 시뮬레이션으로 A/B 테스트 효과를 추정할 수 있어, 중소규모 서비스에서도 데이터 기반 의사결정 가능

- **고도화 로드맵:**

  | 단계 | 내용 | 설명 |
  |------|------|------|
  | Phase 1 (현재 MVP) | 샘플 데이터 기반 데모 | 업종별 가상 데이터로 전체 플로우 체험 |
  | Phase 2 | 실제 고객 데이터 수집 연동 | SDK/API를 통해 고객사의 실제 이벤트 로그를 실시간 수집. 현재 CSV 업로드 방식의 이전 단계에 해당하며, 데이터 수집이 자동화되면 프로파일링과 세그먼트 생성도 지속적으로 갱신됨 |
  | Phase 3 | 실제 테스트 결과 포스트백 반영 | 시뮬레이션 예측 결과와 실제 A/B 테스트 결과를 비교하여 Reaction Model을 자동 보정. 이후 유사한 시나리오의 시뮬레이션 정확도가 점진적으로 향상되는 피드백 루프 구축 |
  | Phase 4 | AI(LLM) 판단이 가미된 결과 도출 | 과거 데이터 기반 통계 모델에 더해, LLM이 시나리오 맥락을 분석하여 Reaction Model 규칙을 자동 생성하고, 결과 리포트에 자연어 인사이트 및 액션 제안을 추가. 시뮬레이션 실행 자체는 통계/확률 모델이 담당하여 재현성을 유지하되, LLM은 "설정"과 "해석" 단계에서 활용 |
  | Phase 5 | 실시간 동기화 트윈 | SDK 연동 + 시간 가중치 + 주기적 재학습으로 디지털 트윈이 실제 유저의 행동 변화를 지속적으로 반영하는 "살아있는 트윈" 구현 |

## 5. 정량적 성과 수치

| 지표 | Before (기존 A/B 테스트) | After (TwinPilot) |
|------|------------------------|-------------------|
| 테스트 소요 시간 | 2~4주 | 수 분 (시뮬레이션 실행 시간) |
| 필요 트래픽 | 수만~수십만 실제 유저 | 0명 (디지털 트윈 기반) |
| 매출 손실 리스크 | 트래픽 분할로 인한 전환율 하락 가능 | 없음 (시뮬레이션이므로 실제 서비스 영향 없음) |
| 세그먼트 분석 | 전체 전환율만 비교 | 자동 세그먼트별 차별 반응 + 최적 시나리오 식별 |
| 테스트 반복 비용 | 매 테스트마다 2~4주 + 트래픽 소모 | 시나리오 변경 후 즉시 재실행 (무제한) |

- **Kiro 활용 개발 성과:**

| 지표 | 수치 |
|------|------|
| 전체 개발 기간 | 약 3일 (기획~배포) |
| 구현된 백엔드 서비스 모듈 | 12개 |
| 구현된 API 엔드포인트 | 17개 (파이프라인 2개 + 개별 15개) |
| 기획 문서 | 요구사항 15개, 설계 문서 (8개 컴포넌트, 21개 정확성 속성), 태스크 20개 |
| 지원 업종 샘플 데이터 | 3개 (패션 이커머스, 금융, OTT 콘텐츠) |
| 결과 리포트 섹션 | 7개 + 자동 인사이트 + 세그먼트 생성 과정 안내 |
| 배포 형태 | Streamlit Cloud (링크 하나로 접속 가능) |
